# Content Summary Skill

将任意 Markdown 内容转换为结构化摘要。

## 概述

| 输出类型 | 文件格式 | 用途 |
|----------|----------|------|
| 结构化摘要 | `*_summary.md` | 核心观点、关键信息提取 |

## 输出目录

所有输出统一保存到 `output/` 目录：

```
output/
├── {date}_{source}_source.md      # 原始内容
└── {date}_{source}_summary.md     # 结构化摘要
```

命名规则：`{YYYY-MM-DD}_{作者或来源}_summary.md`

## 工作流程

```
输入 Markdown
     ↓
分析内容结构
     ↓
提取核心要素
├── 核心主题
├── 关键观点 (3-5个)
├── 支撑证据/数据
├── 引用/链接
└── 行动建议
     ↓
生成结构化摘要 → references/summary-template.md
```

## 输出格式

参考模板：[`references/summary-template.md`](references/summary-template.md)

核心结构：
- **核心观点**: 1-3 个主要论点（可勾选）
- **关键信息**: 表格形式的重要数据
- **行动建议**: 可执行的下一步
- **延伸阅读**: 相关主题链接
- **元数据**: 来源、时间、字数

## 内容分析指南

### 识别核心要素

1. **核心主题**: 内容在讨论什么？用一句话概括
2. **关键观点**: 作者的主要论点是什么？
3. **支撑证据**: 有什么数据、案例、引用支持观点？
4. **实践价值**: 读者可以从中获得什么行动指南？
5. **延伸方向**: 可以进一步探索的相关主题

### 质量标准

- 摘要应该让读者在 30 秒内理解核心内容

## Human Checkpoints

| 环节 | 触发条件 | 用户操作 |
|------|----------|----------|
| 📝 内容确认 | 分析完成后 | 确认核心观点提取是否准确 |
| ✅ 输出确认 | 全部生成后 | 预览并确认质量 |
