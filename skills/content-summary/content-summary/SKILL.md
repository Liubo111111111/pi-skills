# Content Summary Skill

将任意 Markdown 内容转换为结构化摘要。

如果上一步已经成功提取网页或推文正文，不要只回显原文信息；直接产出中文摘要、核心观点和可执行结论。
如果用户要求把结果同步到 Notion、知识库或数据库，交给 `$summary-to-notion`，不要在本 skill 内重复维护 Notion 写入逻辑。

## 概述

| 输出类型 | 文件格式 | 用途 |
|----------|----------|------|
| 结构化摘要 | `*_summary.md` | 核心观点、关键信息提取 |

## 输出目录

所有输出统一保存到 `output/` 目录：

```
output/
├── {date}_{source}_source.md      # 原始内容
└── {date}_{source}_summary.md     # 结构化摘要
```

命名规则：`{YYYY-MM-DD}_{作者或来源}_summary.md`

## 工作流程

```
输入 Markdown
     ↓
分析内容结构
     ↓
提取核心要素
├── 核心主题
├── 关键观点 (3-5个)
├── 支撑证据/数据
├── 引用/链接
└── 行动建议
     ↓
生成结构化摘要 → references/summary-template.md
```

## 输出格式

参考模板：[`references/summary-template.md`](references/summary-template.md)

核心结构：
- **核心观点**: 1-3 个主要论点（可勾选）
- **关键信息**: 表格形式的重要数据
- **行动建议**: 可执行的下一步
- **延伸阅读**: 相关主题链接
- **元数据**: 来源、时间、字数

## 飞书消息发送（自动执行）

生成摘要后，**自动发送飞书消息**给用户，确保内容完整显示。

### 发送格式

飞书消息使用简化版 Markdown，确保完整显示：

```markdown
## 📌 {标题}

### 核心观点
- ✅ {观点1}
- ✅ {观点2}
- ✅ {观点3}

### {章节1}
{内容}

### {章节2}
{表格或列表}

---
来源: @{作者} | 提取时间: {YYYY-MM-DD}
```

### 发送时机

- 摘要生成完成后 → 自动发送
- 无需用户确认（除非用户明确要求先确认）

### 发送工具

使用 `message` 工具发送飞书 DM：

```json
{
  "action": "send",
  "channel": "feishu",
  "target": "user:{open_id}",
  "message": "{简化版摘要内容}"
}
```

**注意**：
- 从上下文获取用户 open_id（飞书 DM 场景）
- 消息内容转 JSON 时需正确转义换行和特殊字符
- 超长内容拆分为多条消息发送

## 内容分析指南

### 识别核心要素

1. **核心主题**: 内容在讨论什么？用一句话概括
2. **关键观点**: 作者的主要论点是什么？
3. **支撑证据**: 有什么数据、案例、引用支持观点？
4. **实践价值**: 读者可以从中获得什么行动指南？
5. **延伸方向**: 可以进一步探索的相关主题

### 质量标准

- 摘要应该让读者在 30 秒内理解核心内容

## Human Checkpoints

| 环节 | 触发条件 | 用户操作 |
|------|----------|----------|
| 📝 内容确认 | 分析完成后 | ~~确认核心观点提取是否准确~~ → 自动发送飞书 |
| ✅ 输出确认 | 全部生成后 | ~~预览并确认质量~~ → 自动发送完成 |

**注意**：摘要生成后自动发送飞书消息，用户要求显示完整内容时走此流程。
