# 闪卡生成模式参考

## 格式规范

兼容 Obsidian Spaced Repetition 插件格式：

```markdown
Q: {问题}
A: {答案}
```

使用 `---` 分隔不同闪卡。

## 闪卡类型

### 1. 定义卡 (Definition)

从核心概念生成，测试基本理解。

**模式**:
```
Q: 什么是 {概念}?
A: {概念} 是 {定义}。{可选：一句话补充说明}
```

**示例**:
```
Q: 什么是 Programmatic Tool Calling?
A: Programmatic Tool Calling 是 Claude 的一个功能，可以自动编写代码来调用工具，在沙箱中运行后返回结果给模型。
```

**识别方法**:
- 新术语、专有名词
- 被解释或定义的概念
- 技术名称、产品名

### 2. 事实卡 (Fact)

从关键数据生成，测试记忆。

**模式**:
```
Q: {具体问题}?
A: {精确答案}。
```

**示例**:
```
Q: Programmatic Tool Calling 可以减少什么?
A: 可以减少延迟 (latency) 和 token 消耗 (token consumption)。
```

**识别方法**:
- 数字、百分比、金额
- 具体的优缺点列表
- 明确的因果关系

### 3. 理解卡 (Understanding)

从观点生成，测试深度理解。

**模式**:
```
Q: 为什么 {现象/观点}?
A: 因为 {原因}。{可选：进一步解释}
```

**示例**:
```
Q: 为什么 Programmatic Tool Calling 能减少 token 消耗?
A: 因为它在沙箱中直接运行代码执行工具调用，而不是让模型通过多轮对话来完成，减少了往返次数。
```

**识别方法**:
- 作者的论证和推理
- 解释性内容
- 因果分析

### 4. 应用卡 (Application)

从行动建议生成，测试实践能力。

**模式**:
```
Q: 如何 {应用/实践}?
A: {具体步骤或方法}。
```

**示例**:
```
Q: 如何使用 Programmatic Tool Calling?
A: 在 Claude API 中启用 beta 功能，定义工具 schema，让 Claude 自动生成调用代码在沙箱中执行。
```

**识别方法**:
- 教程、步骤说明
- 最佳实践
- 使用建议

### 5. 比较卡 (Comparison)

用于有对比内容时，测试区分能力。

**模式**:
```
Q: {A} 和 {B} 的主要区别是什么?
A: {A} {特点1}，而 {B} {特点2}。
```

**识别方法**:
- 明确的对比内容
- 优缺点列表
- 不同方案/技术

## 数量配置

### Minimal (3-5 张)

只生成最核心的卡片：
- 1 张定义卡（核心概念）
- 1-2 张事实卡（关键数据）
- 1-2 张理解卡（主要观点）

### Standard (8-12 张)

平衡覆盖：
- 2-3 张定义卡
- 2-3 张事实卡
- 2-3 张理解卡
- 2-3 张应用卡

### Comprehensive (15-20 张)

完整覆盖：
- 3-4 张定义卡（所有重要概念）
- 4-5 张事实卡（所有关键数据）
- 4-5 张理解卡（所有观点）
- 4-6 张应用卡（所有建议）

## 闪卡模板

```markdown

> 来源: {URL}
> 生成时间: {日期}
> 卡片数量: {N} 张

---

## 核心概念

Q: {问题1}
A: {答案1}

---

Q: {问题2}
A: {答案2}

---

## 关键事实

A: {答案3}

---

## 深度理解

Q: {问题4}
A: {答案4}

---

## 实践应用

Q: {问题5}
A: {答案5}

---

*由 content-summary skill 生成*
```

## 质量检查清单

- [ ] 问题是否具体明确？
- [ ] 答案是否简洁准确？
- [ ] 是否避免了模糊的表述？
- [ ] 是否覆盖了核心知识点？
- [ ] 问题之间是否有重复？
- [ ] 难度分布是否合理？

## 常见问题

### 内容太少

如果源内容很短（如一条推文），闪卡数量可以少于配置的最小值。此时：
- Minimal: 2-3 张
- Standard: 3-5 张
- Comprehensive: 5-8 张

### 内容太杂

如果内容涉及多个不相关主题，建议：
1. 分别为每个主题生成闪卡集
2. 或只关注最核心的主题

### 已有相关知识

对于延伸阅读或相关主题，可以生成"桥接卡"：
```
Q: {新概念} 与 {已知概念} 有什么关系?
A: {关系说明}
```
