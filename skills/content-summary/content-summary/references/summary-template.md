# 结构化摘要模板

## 模板结构

```markdown
# {标题} - 知识摘要

## 核心观点

- [ ] **观点1**: {一句话描述}
- [ ] **观点2**: {一句话描述}
- [ ] **观点3**: {一句话描述}

## 关键信息

| 类型 | 内容 |
|------|------|
| 人物/组织 | {相关人物或组织} |
| 关键数据 | {重要数字、统计} |
| 技术/工具 | {涉及的技术或工具} |
| 时间节点 | {重要日期或时间线} |

## 详细说明

### {观点1标题}

{详细解释，2-3 句话}

**证据**: {支撑这个观点的证据}

### {观点2标题}

{详细解释}

**证据**: {支撑证据}

## 行动建议

1. **立即可做**: {具体行动}
2. **短期目标**: {1-2周内可完成}
3. **长期探索**: {持续关注的方向}

## 延伸阅读

- [[{相关主题1}]]
- [[{相关主题2}]]
- [{外部链接}]({URL})

## 元数据

| 属性 | 值 |
|------|------|
| 来源 | {URL 或文档名} |
| 作者 | {原作者} |
| 提取时间 | {YYYY-MM-DD} |
| 原文字数 | {约 X 字} |
| 内容类型 | {文章/推文/对话/...} |

---
*由 content-summary skill 生成*
```

## 填写指南

### 核心观点提取

1. 阅读全文，识别作者反复强调的点
2. 寻找转折词后的内容（但是、然而、关键是）
3. 注意结论段落的总结
4. 限制在 3-5 个观点，过多则需要合并

### 关键信息分类

| 类型 | 识别方法 |
|------|----------|
| 人物/组织 | 专有名词、@ 提及、公司名 |
| 关键数据 | 数字、百分比、金额 |
| 技术/工具 | 产品名、技术术语、API |
| 时间节点 | 日期、版本号、里程碑 |

### 行动建议生成

根据内容类型生成不同风格的建议：

- **技术文章**: 尝试实践、阅读文档、对比方案
- **观点评论**: 反思验证、收集更多视角、形成自己观点
- **新闻资讯**: 持续关注、评估影响、调整策略

### 延伸阅读

1. 从内容中提取的链接
2. 相关技术或概念的官方文档
3. 可以用 [[]] 格式链接到 Obsidian 中的其他笔记
